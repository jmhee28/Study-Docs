# created_at, updated_at 필요 이유
- 생성/수정 시점을 기록해 데이터 변경 이력을 관리하기 위해 필요
- 문제 발생 시점 추적 및 감사(Audit) 로그 근거로 활용
- 최신 데이터 판별, 동기화 충돌 해결 등 비즈니스 규칙에 사용
- 데이터 품질 점검(오래된 데이터, 미갱신 데이터) 기준으로 사용
- 성능 튜닝(인덱스, 보관/아카이브 정책) 기준 마련에 도움

# deleted_at 필요 이유
- 소프트 삭제(논리 삭제)로 복구 가능성을 확보
- 삭제 시점 기록으로 감사(Audit) 및 추적 근거 제공
- 운영 실수/정책 오류 발생 시 빠른 복구 지원
- 실데이터와 삭제 데이터 분리로 통계 왜곡 방지
